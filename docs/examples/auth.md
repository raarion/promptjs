## Tujuan / Goal

Membangun flow autentikasi lengkap:

1. Halaman utama dilindungi auth guard ŌĆö pengguna tanpa token di-redirect ke login
2. Login form dengan validasi client-side
3. Setelah login sukses ŌåÆ simpan token + redirect ke dashboard
4. Logout ŌåÆ hapus token + redirect ke login

Build a complete authentication flow:

1. Main page protected by auth guard ŌĆö users without a token are redirected to login
2. Login form with client-side validation
3. After successful login ŌåÆ save token + redirect to dashboard
4. Logout ŌåÆ remove token + redirect to login

## Bagian 1: Auth Guard di Halaman Utama / Part 1: Auth Guard on Main Page

```pjs
---
judul: "PromptJS Dashboard"
butuhAuth: benar
redirect: /login
token: localStorage
tokenKey: auth_token
---

Halaman DashboardApp:
    # ... konten dashboard ...
```

### Compiler emit (apa yang terjadi di balik layar) / Compiler emit (what happens behind the scenes)

```js
(function() {
    // Auth guard (generated by PromptJS compiler)
    const __token = localStorage.getItem("auth_token");
    if (!__token) {
        window.location.href = "/login";
        return;  // Stop ŌĆö halaman tidak di-render
    }
    // ... page content ...
})();
```

**Yang terjadi:** Sebelum konten halaman dieksekusi, compiler menyuntikkan pengecekan token. Jika `localStorage.auth_token` kosong ŌåÆ redirect ke `/login`. Jika ada ŌåÆ lanjut render halaman.

**What happens:** Before the page content executes, the compiler injects a token check. If `localStorage.auth_token` is empty ŌåÆ redirect to `/login`. If present ŌåÆ continue rendering the page.

## Bagian 2: Login Form / Part 2: Login Form

```pjs
---
judul: "Masuk"
---

Halaman Login:
    data nama = ""
    data sandi = ""
    data pesan = ""

    Buat div.login-card:
        Buat h2: $judul
        Buat span.subtitle: "Masuk ke dashboard anda"

        Buat div.form-group:
            Buat label: "Nama Pengguna"
            Buat masukan#nama:
                placeholder = "admin"
                on_diketik = simpan document.querySelector("#nama").value ke nama

        Buat div.form-group:
            Buat label: "Kata Sandi"
            Buat masukan#sandi:
                type = "password"
                placeholder = "ŌĆóŌĆóŌĆóŌĆóŌĆóŌĆó"
                on_diketik = simpan document.querySelector("#sandi").value ke sandi

        Buat div.error-msg#pesan-error:
            Saat pesan:
                Buat span:
                    pesan

        Buat tombol.login-btn#btn-login: "Masuk"
            Ketika diklik:
                Jika nama === "admin":
                    simpan "admin" ke localStorage.auth_token
                    simpan "Administrator" ke localStorage.nama_pengguna
                    simpan "admin" ke localStorage.__peran
                    arahkan "dashboard.html"
                Lainnya:
                    simpan "ŌÜĀ’ĖÅ Nama atau sandi salah" ke pesan

        Buat div.login-footer:
            Buat span: "Gunakan "
            Buat code: "admin"
            Buat span: " untuk mencoba"
```

### Yang terjadi / What's happening

1. **`data nama`, `data sandi`, `data pesan`** ŌĆö tiga state reaktif: input nama, input sandi, dan pesan error.
2. **`on_diketik`** ŌĆö setiap ketikan menyimpan nilai input ke state. Ini real-time binding tanpa framework.
3. **`Ketika diklik`** ŌĆö saat tombol ditekan, cek kredensial. Jika `nama === "admin"` ŌåÆ simpan token + data sesi ke localStorage, lalu `arahkan` ke dashboard.
4. **`Lainnya`** ŌĆö jika kredensial salah, tampilkan pesan error. `Saat pesan:` membuat pesan muncul secara reaktif.
5. **Login "admin"** ŌĆö ini demo auth client-side. Di production, ganti dengan `Ambil dari` POST ke API backend.

## Bagian 3: Logout / Part 3: Logout

Dari halaman mana pun yang dilindungi (Dashboard, Profil, Pengaturan):

```pjs
Buat tombol: "Keluar"
    Ketika diklik:
        hapus localStorage.auth_token
        hapus localStorage.nama_pengguna
        hapus localStorage.__peran
        arahkan "login.html"
```

**Yang terjadi:** `hapus localStorage.x` di-lower menjadi `localStorage.removeItem("x")`. Setelah token dihapus, `arahkan` ke login. Jika pengguna mencoba akses dashboard lagi, auth guard akan memblokir.

**What happens:** `hapus localStorage.x` is lowered to `localStorage.removeItem("x")`. After the token is removed, `arahkan` to login. If the user tries to access the dashboard again, the auth guard blocks them.

## Bagian 4: Role Check (Bonus) / Part 4: Role Check

```pjs
---
butuhAuth: benar
redirect: "/login"
token: localStorage
tokenKey: auth_token
peran: admin
---
```

Dengan `peran: admin`, compiler menambahkan pengecekan role **setelah** pengecekan token:

```js
// Setelah cek token:
const __peran = localStorage.getItem("__peran");
if (__peran !== "admin") {
    window.location.href = "/login";
    return;
}
```

## Pola yang Direkomendasikan / Recommended Patterns

### Ō£ģ Real auth: Ganti cek client-side dengan API call

```pjs
Ketika diklik:
    Ambil dari "/api/auth/login":
        metode = "POST"
        isi = { nama: nama, sandi: sandi }
        berhasil:
            simpan __data.token ke localStorage.auth_token
            arahkan "dashboard.html"
        gagal:
            simpan "Kredensial salah" ke pesan
```

### Ō£ģ Token di header fetch

```pjs
Ambil dari "/api/profil":
    header = { Authorization: "Bearer " + localStorage.auth_token }
    berhasil:
        simpan __data ke profil
```
