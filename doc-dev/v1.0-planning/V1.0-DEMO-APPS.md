# PromptJS — v1.0.0 Demo Apps Specification

> Detailed specification for the two demo applications required by FASE 5.
> These apps demonstrate PromptJS capabilities and serve as real-world examples.

---

## 1. TODO APP

### Purpose

Demonstrate basic CRUD operations, reactive state, event handling, and localStorage persistence — the "hello world" of interactive apps.

### File Structure

```
examples/todo-app/
├── index.pjs          — Main page with todo list
├── data/
│   └── initial.json   — Seed data (optional)
└── README.md          — App description
```

### Feature Specification

**State:**
```pjs
data daftar = []
data filter = "semua"       # semua / aktif / selesai
data inputTeks = ""
```

**UI Elements:**
1. Header with app title
2. Input field for new todo + "Tambah" button
3. Filter buttons: Semua / Aktif / Selesai
4. Todo list with checkbox, text, delete button
5. Counter: "X item tersisa"

**Interactions:**
- `Ketika diklik` on "Tambah" → add new item to `daftar`
- `Ketika diubah` on checkbox → toggle item's `selesai` flag
- `Ketika diklik` on delete → remove item from `daftar`
- `Ketika diklik` on filter → update `filter` variable
- `Saat daftar berubah` → persist to localStorage + update counter

**Auth:** None (public app)

**Estimated PJS code:** ~60 lines

### Implementation Notes

- Use `Ulangi untuk item dari $daftar` to render list
- Use `Jika` to filter items based on `filter` state
- Use `simpan` + `JSON.stringify` / `JSON.parse` for localStorage
- Use `tambahkan` / `kurangi` for counter updates

## 2. DASHBOARD APP

### Purpose

Demonstrate the full v0.9.9+ feature set: SPA routing, auth guard, peran role check, data fetching, and lifecycle hooks.

### File Structure

```
examples/dashboard-app/
├── index.pjs              — Main entry (router + auth config)
├── pages/
│   ├── login.pjs          — Login form
│   ├── dashboard.pjs      — Stats & counters (auth required, peran: admin)
│   ├── profil.pjs         — User profile (auth required)
│   └── pengaturan.pjs     — App settings (auth required)
└── README.md
```

### Feature Specification

**index.pjs (Router + Auth):**
```pjs
butuhAuth: benar
redirect: "/login"
token: localStorage
tokenKey: auth_token
router: benar
---
Halaman App:
    Buat nav#navigasi:
        Buat span.logo: "Dashboard"
        Buat tombol[data-halaman="dashboard"]: "Dashboard"
        Buat tombol[data-halaman="profil"]: "Profil"
        Buat tombol[data-halaman="pengaturan"]: "Pengaturan"
        Buat tombol#keluar: "Keluar"
    Buat div#konten:
        Buat h2: "Memuat..."
```

**login.pjs:**
```pjs
---
judul: "Masuk"
---
Halaman Login:
    data nama = ""
    data sandi = ""
    data pesan = ""

    Buat div.kartu-login:
        Buat h2: "Masuk ke Dashboard"
        Buat formulir:
            Buat label: "Nama Pengguna"
            Buat input#nama[placeholder="admin"]: ""
            Buat label: "Kata Sandi"
            Buat input#sandi[type="password"]: ""
            Buat span.pesan: $pesan
            Buat tombol#masuk: "Masuk"
                Ketika diklik:
                    Jika $nama === "admin":
                        simpan "dummy-token" ke localStorage.auth_token
                        simpan "admin" ke localStorage.__peran
                        arahkan ke "/dashboard"
                    Lainnya:
                        simpan "Nama atau sandi salah" ke pesan
```

**dashboard.pjs:**
```pjs
---
judul: "Dashboard"
peran: admin
---
Halaman Dashboard:
    data hitung = 0
    data pengunjung = 0

    Ketika dipasang:
        simpan 42 ke pengunjung

    Buat h2: "Dashboard Admin"
    Buat div.stats:
        Buat div.kartu-stat:
            Buat span.angka: $hitung
            Buat span.label: "Counter"
            Buat tombol#tambah: "+"
                Ketika diklik:
                    tambahkan 1 ke hitung
            Buat tombol#kurang: "-"
                Ketika diklik:
                    kurangi 1 dari hitung
        Buat div.kartu-stat:
            Buat span.angka: $pengunjung
            Buat span.label: "Pengunjung"
```

**profil.pjs:**
```pjs
---
judul: "Profil"
---
Halaman Profil:
    data nama = ""
    data peran = ""

    Ketika dipasang:
        simpan localStorage.getItem("auth_token") ke nama
        simpan localStorage.getItem("__peran") ke peran

    Buat h2: "Profil Pengguna"
    Buat div.profil:
        Buat label: "Token"
        Buat span: $nama
        Buat label: "Peran"
        Buat span: $peran
```

**pengaturan.pjs:**
```pjs
---
judul: "Pengaturan"
---
Halaman Pengaturan:
    Buat h2: "Pengaturan"
    Buat div.pengaturan:
        Buat label: "Tema"
        Buat pilihan#tema:
            Buat opsi[value="terang"]: "Terang"
            Buat opsi[value="gelap"]: "Gelap"
        Buat tombol#keluar: "Keluar dari Akun"
            Ketika diklik:
                hapus localStorage.auth_token
                hapus localStorage.__peran
                arahkan ke "/login"
```

### Interactions

1. **Auth flow:** User visits → no token → redirect to /login → login → set token + peran → navigate to dashboard
2. **Role check:** dashboard has `peran: admin` → if localStorage.__peran !== 'admin' → redirect
3. **SPA routing:** Nav buttons switch pages without reload
4. **Lifecycle:** `Ketika dipasang` loads data when page enters
5. **Logout:** `hapus localStorage.auth_token` + redirect to /login

### Implementation Notes

- This is a richer version of the `fullstack` init template
- The login uses dummy authentication (no backend)
- Peran is stored in `localStorage.__peran` (current convention)
- Logout uses `hapus` lowering → `removeItem()`
- The dashboard counter demonstrates reactive state + event handlers

## 3. BUILD & SERVE

Both demo apps should be buildable and servable:

```bash
# Dev
pjs serve examples/todo-app/

# Build
pjs build examples/todo-app/ --adapter static

# SPA build
pjs build examples/dashboard-app/ --adapter static --spa
```
