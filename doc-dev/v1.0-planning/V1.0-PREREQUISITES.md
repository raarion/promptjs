## 2. DECISIONS REQUIRED BEFORE v1.0.0

### 2.1 Package Name

**Options:**
- `promptjs` — simplest, may conflict with existing npm packages
- `@promptjs/cli` — scoped package, more professional
- `pjs` — short but likely taken
- `@promptjs/core` — if separating CLI from compiler

**Decision needed:** Search npm registry for availability, choose name.

### 2.2 Peran Storage Key

Currently the peran check reads from `localStorage.getItem('__peran')`. This is hardcoded.

**Options:**
- Keep `__peran` as internal convention
- Add `peranKey` directive (like `tokenKey`)
- Read peran from decoded JWT token

**Recommendation:** For v1.0.0, keep `__peran` as convention. Add `peranKey` in v1.1 if needed.

### 2.3 Multi-Peran Support

Current: `peran: admin` → single role check. What about multiple allowed roles?

**Options:**
- `peran: admin` → single role (current)
- `peran: admin,editor` → comma-separated roles → `__allowedPeran.includes(__peran)`
- `peran: [admin, editor]` → array syntax (more complex parsing)

**Recommendation:** For v1.0.0, keep single role. Multi-peran can be v1.1 feature.

### 2.4 Tutorial Language

**Options:**
- Indonesia only
- English only
- Bilingual (ID + EN side by side)

**Recommendation:** Bilingual with Indonesia as primary, matching the DSL's bilingual principle.

### 2.5 Breaking Changes Before Semver Lock

Any API surface that should change before 1.0.0 locks semver?

- `__peran` internal storage key → may change
- Auth guard IIFE pattern → stable, no change needed
- Compiler output format → stable since v0.5
- CLI command names → stable

**Recommendation:** No breaking changes needed. The API surface is stable.

## 4. CI/CD HARDENING

### Branch Protection Rules

- `main` branch: require PR review, require CI pass, no force push
- Release tags: protected, only from main

### Release Workflow (GitHub Actions)

```yaml
name: Release
on:
  push:
    tags: ['v*']
jobs:
  release:
    steps:
      - checkout
      - setup node
      - npm ci
      - npm test
      - npm publish --access public
```

### npm Publish Checklist

1. Verify package.json `name` is available on npm
2. Verify `files` field includes only necessary files (no tests, no doc-dev)
3. Verify `main` and `bin` fields are correct
4. `npm publish --dry-run` to test
5. `npm publish --access public`
