> **Wave**: D2.1

Test suite untuk error/warning codes yang di-defer dari D2 karena membutuhkan setup kompleks atau bug fix tambahan.

## Bug fix yang terkait dengan D2.1

1. **Engine** — Resolver warnings (W3002, W3003, W4003) sebelumnya silently discarded. Engine hanya collect analyzer warnings dari stage 4, tapi tidak collect resolver warnings dari stage 3. Fixed: engine sekarang forward `resolveResult.warnings` ke `this.warnings`.

2. **Parser** — Type hint parsing (`data x: angka = "teks"`) sebelumnya hardcoded `typeHint = null` dengan komentar "v0.1 doesn't support type hints yet". Padahal lexer sudah emit token type hint dengan benar. Fixed: parser sekarang parse `: typeHint` setelah nama variabel sebelum `= init`.

3. **Analyzer** — `visitSaatStatement` sebelumnya akses `node.target` sebagai string (`this.lookupSymbol(node.target)`), padahal `node.target` adalah AST node Identifier. `String(node.target)` menghasilkan `"[object Object]"` sehingga lookup gagal dan W4104 tidak pernah muncul. Fixed: extract nama dari Identifier/MemberExpression (same fix pattern as resolver D1).

### Detail Test Case

### 1. `E4201` — dependency cycle pada data turunan ✅

**Kategori**: Error codes (complex setup)

**Input**:
```pjs
turunan a = b tambah 1
turunan b = a tambah 1
```

**Expected**: E4201 (dependency cycle) di-throw oleh analyzer setelah `buildDependencyGraph` mendeteksi cycle `a → b → a`.

**Catatan**: Setup ini membutuhkan understanding bahwa resolver melakukan hoisting via `gatherGlobals` — kedua `turunan` dideklarasikan sebelum traversal, sehingga keduanya visible saat dependency graph dibangun. Tanpa hoisting, `turunan a = b tambah 1` akan gagal E3001 karena `b` belum dideklarasikan.

### 1. `W4003` — deklarasi tetap tanpa nilai awal ✅

**Input**:
```pjs
tetap x
```

**Catatan**: `tetap` (const) tanpa nilai awal tidak masuk akal — konstanta harus diinisialisasi. Analyzer emit W4003.

---

### 2. `W3002` — variabel shadowing scope luar ✅

**Input**:
```pjs
data x = 1
Fungsi halo():
    data x = 2
    kembalikan x
```

**Catatan**: `data x = 2` di dalam Fungsi body membuat `x` baru di scope fungsi, men-shadow `x` di global scope. Resolver emit W3002. Test ini butuh FungsiDeclaration (yang baru bisa compile setelah D1 bug fix untuk word operator collision).

---

### 3. `W3003` — watcher target bukan data reaktif (resolver) ✅

**Input**:
```pjs
tetap x = 5
Saat x:
    "berubah"
```

**Catatan**: `tetap` (const) tidak reaktif. `Saat x:` men-watch variabel non-reaktif — watcher tidak akan pernah terpicu. Resolver emit W3003. Bug fix D2.1: resolver warnings sebelumnya tidak di-forward oleh engine ke result.

---

### 4. `W4001` — type hint tidak cocok dengan nilai ✅

**Input**:
```pjs
data x: angka = "teks"
```

**Catatan**: Type hint `angka` (number) tidak cocok dengan nilai `"teks"` (string). Analyzer's `checkTypeHint` emit W4001. Bug fix D2.1: parser sebelumnya hardcoded `typeHint = null`, sehingga type hint tidak pernah masuk ke AST dan W4001 tidak pernah fire.

---

### 5. `W4104` — watcher target bukan data reaktif (analyzer) ✅

**Input**:
```pjs
tetap x = 5
Saat x:
    "berubah"
```

**Catatan**: Warning yang sama dengan W3003, tapi dari analyzer (bukan resolver). Bug fix D2.1: analyzer `visitSaatStatement` sebelumnya akses `node.target` sebagai string, padahal itu AST node. Fixed dengan extract nama dari Identifier/MemberExpression.

## 3. Feature gap

### Ringkasan

| # | Test ID | Nama | Status | Alasan |
|---|---------|------|--------|--------|
| 1 | `W4102` | W4102: simbol ditulis tapi tidak pernah dibaca | ⏭️ | Needs `simpan` keyword |
| 2 | `W4103` | W4103: data reaktif dimutasi tapi tidak dibaca | ⏭️ | Needs `simpan` keyword |
| 3 | `E5001` | E5001: node AST tidak didukung compiler | ⏭️ | Needs compiler validation |

### Catatan

- **W4102 dan W4103**: Keduanya butuh keyword `simpan` untuk membuat explicit write ke variabel. Tanpa `simpan`, writes hanya terjadi via init (`data/tetap/ubah = value`) dan property assignment (`x = value` di Buat body, yang di-parse sebagai atribut elemen, bukan SimpanStatement). Implementasi keyword `simpan` akan otomatis memungkinkan kedua warning ini dipicu.

- **E5001**: Compiler saat ini silently ignores unknown AST node types (via `accept()` → `genericVisit` → no-op). E5001 hanya fire jika `compiler.compile()` throws exception, yang tidak terjadi dengan kode saat ini. Untuk memicu E5001, compiler perlu validasi node type dan throw untuk node yang tidak dikenali.
